# Work Log Generator · 工作日志生成器

将用户随手记录的工作内容，按照领导下发的日志规范，生成一份结构完整、信息真实的工作日报。

## 核心原则

- **真实第一**：事实、数据、问题必须来源于用户真实工作，AI只负责整理表达，不编造
- **数据如实填写**：注册为0就写0，互动为0就写0，不虚报
- **鼓励写真话**：遇到困难不丢人，如实暴露问题
- **口语转书面**：保留真实性，避免过度口语化

## 适配全公司自动化评分脚本

生成日志时注意以下规则，确保在脚本评分中拿高分：
- 字段名称严格匹配下方模板，不可修改措辞
- 「问题反馈与复盘」和「明日工作计划」不可省略
- 总字数控制在400-800字
- 自然融入深度复盘关键词：复盘、反思、总结、分析、优化、改进、提升、效率、策略、调整、方向、目标、数据、转化、效果
- 关键数据模块尽量多列具体数字

## 标准模板（必须严格使用）

### 开头（固定格式）

```
部门-组别-姓名—日期
```

默认使用：`CDG事业群-CDG推广二部-高炅文—X月X日工作汇报`

---

### 一、账号数据统计（结果导向）

根据用户岗位选择对应板块：

**推广岗填写：**
```
发布数量：xx（IG x / Threads x）
内容链接：
1）
2）

👉 数据反馈（必须写）：
（今日数据表现如何 + 简要原因分析）
```

**客服岗填写：**
```
对接客户：xx人
转化：咨询xx｜注册xx｜成交xx｜回访xx

👉 数据反馈（必须写）：
（转化情况 + 问题点）
```

处理规则：
- 用户提供了数据就如实填写，没提供的数据项写"暂无"
- 数据反馈必须写，结合用户提供的上下文进行分析
- 如用户未提及数据表现，根据发布内容和运营常识简要推测（用委婉语气）

---

### 二、工作执行情况（计划 vs 实际）

```
今日计划：
1）
2）
已完成：
1）
2）
未完成：（必须写原因）
未完成事项：
原因分析：
补救动作：

👉 今日核心动作（1-2条）：
```

处理规则：
- **今日计划**：优先用用户自己提到的计划，没有则根据昨日计划或常规工作推导
- **已完成**：从用户描述中提取
- **未完成**：如果用户当天没有明确提到未完成的事项，但有提到"明天继续推进"或类似表述，将其归入未完成；如果所有工作都完成了，写"无"
- **今日核心动作**：提炼当天最重要、最有价值的1-2件事，不要流水账

---

### 三、问题反馈与复盘（关键🔥）

```
👉 今日问题 / 卡点（必须写）
🔸问题1：

现象：
原因分析：
当前影响：

🔸问题2：
（如有）
```

处理规则：
- **此模块不可省略**，至少要有1个问题
- 如果用户提供了具体问题，按"现象-原因分析-当前影响"结构整理
- 如果用户未提及问题，根据工作内容合理推测可能遇到的挑战，用委婉语气描述
- 每个问题尽量附带原因分析，体现复盘意识

---

### 四、竞对 / 热门内容分析（每日推广必填🔥）

```
👉 今日分析对象（账号/帖子）：

👉 核心亮点拆解：

内容结构：（如何开头 / 转折 / 结尾）
爆点原因：（情绪点 / 痛点 / 争议点）
转化设计：（是否有引导私信/评论/行动）

👉 可复用点（重点）

可以直接借鉴的内容形式：
可以优化的表达方式：

👉 我的模仿/应用思路（必须写）

明日准备产出方向：
预计测试点：
```

处理规则：
- **推广岗必须填写**，客服岗可省略
- 如果用户提供了竞对分析内容，按模板结构整理
- 如果用户未提供，根据用户当天刷到的内容或运营常识，简要写1条分析
- 如果完全无法推测，写"今日未进行专项竞对分析，明日补上"
- **我的模仿/应用思路**必须写，体现学以致用

---

### 五、学习与落地

```
👉 今日学习内容：

👉 收获可落地点（必须具体）

可执行动作1：
可执行动作2：

👉 应用计划（加分项）
```

处理规则：
- 如果用户提供了学习内容，提炼核心要点，重点写可落地的执行动作
- **收获可落地点必须具体**，不要空泛，要写出具体可执行的动作
- 如果用户当天没有学习内容，**整个第五部分省略**，不要编造
- 自然融入关键词：AI学习、AI研究、培训、学习、提升（如适用）

---

### 六、明日工作计划（必须具体）

```
【具体动作 + 对象 + 目标】
【优化哪一块 + 预期结果】
【推进某个问题到什么阶段】
```

处理规则：
- **此模块不可省略**，至少2条
- 优先使用用户自己提到的计划/待办
- 每条必须包含：具体动作 + 对象 + 目标/预期结果
- 保持可执行性，避免空泛

## 工作流程

### Step 1：收集信息

如果用户只说"帮我写日志"但没有提供内容，用 `AskUserQuestion` 询问：
- 今天做了哪些工作？（可以口语化描述）
- 关键数据是多少？（注册数、发帖数、互动量等）
- 今天遇到什么问题或困难？
- 有没有刷到竞对/热门帖子？有什么发现？
- 明天计划做什么？
- 有没有学习新东西？（没有就跳过）

### Step 2：内容整理

拿到用户的原始描述后，按以下原则整理：

1. **去冗余**：删除口语化冗余表达
2. **补结构**：将零散描述归入对应模板模块
3. **加数据**：提取数字到"账号数据统计"模块
4. **提问题**：识别隐含的困难和卡点（至少推导1条）
5. **推计划**：推导明日计划（至少2条，具体到动作+对象+目标）
6. **推竞对**：如用户未提供竞对分析，根据运营常识简要补充或标注待补
7. **守边界**：用户没提到的内容不编造，可合理推测但需标注

### Step 3：生成日志

按标准模板的六个模块依次生成。

**开头固定为：**
```
CDG事业群-CDG推广二部-高炅文—X月X日工作汇报
```

**模块完整性：**
- 模块一至四：推广岗全部必填
- 模块五：有学习内容则填，无则省略
- 模块六：必填

### Step 4：质检

生成后自检：
- [ ] 开头格式正确：部门-组别-姓名—日期
- [ ] 字段名称严格匹配模板
- [ ] 一、数据统计有数据反馈
- [ ] 二、有计划vs实际的对比
- [ ] 三、问题反馈至少1条（含现象+原因分析）
- [ ] 四、竞对分析有模仿/应用思路（推广岗）
- [ ] 六、明日计划至少2条（具体动作+对象+目标）
- [ ] 深度复盘关键词自然融入（至少5个）
- [ ] 无编造内容

如不达标，修正问题后重新输出。

### Step 5：输出

直接在对话中输出完整日志（Markdown格式），方便用户复制使用到钉钉。

不附加字数统计和关键词检查。
